# 숨길(Soomgil) UI 기획 및 프롬프트 제안서

아래는 기획서의 핵심 콘셉트인 **“스와이프로 취향을 모으고, 지도 위에서 함께 그리는 그룹 여행 설계 플랫폼”**을 바탕으로 만든 **웹 서비스 UI 제작용 프롬프트 제안**입니다. 서비스는 스와이프 기반 취향 수집, 그룹 슈퍼라이크 공유, 지도 캔버스 경로 설계, AI 추천, 여행 커뮤니티 피드를 핵심 기능으로 합니다.

이 문서는 구현 지침의 source of truth가 아니라 기획 방향과 화면 아이디어를 확인하기 위한 참고 문서입니다. 구현 시에는 `README.md`의 문서 라우팅을 따라 `AGENTS.md`, `.agent/docs/` 하위 가이드, `frontend/`의 실제 Vue 컴포넌트와 스타일을 우선합니다.

---

## 1. 메인 랜딩 페이지 UI 프롬프트

```text
20~30대 그룹 여행자를 위한 취향 기반 여행 협업 플랫폼의 웹 랜딩 페이지 UI를 디자인해줘.

서비스 콘셉트는 “스와이프로 취향을 모으고, 지도 위에서 함께 그리는 그룹 여행 설계 플랫폼”이다.

전체 분위기는 젊고 감각적이며, SNS와 여행 앱에 익숙한 사용자를 대상으로 한다. 
밝고 경쾌한 컬러를 사용하되, 지도·여행·커뮤니티 느낌이 자연스럽게 드러나야 한다.

구성 요소:
1. 상단 네비게이션
   - 로고
   - 서비스 소개
   - 주요 기능
   - 커뮤니티
   - 시작하기 버튼

2. 히어로 섹션
   - 큰 카피: “친구들과 취향을 모아, 우리만의 여행 루트를 그려보세요”
   - 서브 카피: “스와이프로 여행지를 고르고, 지도 위에서 함께 경로를 설계하는 그룹 여행 플랫폼”
   - CTA 버튼: “여행방 만들기”, “서비스 둘러보기”
   - 우측에는 관광지 카드 스와이프 화면과 지도 경로 화면이 겹쳐 보이는 목업

3. 문제 제기 섹션
   - “어디 갈까?”를 반복하는 그룹 여행의 불편함
   - 흩어진 여행 정보
   - 인기 관광지 중심 추천
   - 일정 조율 피로

4. 핵심 기능 섹션
   - 스와이프 관광지 탐색
   - 슈퍼라이크 기반 그룹 취향 공유
   - 지도 캔버스 경로 설계
   - AI 추천 여행지
   - 여행 루트 커뮤니티

5. 하단 CTA
   - “이번 여행, 취향부터 맞춰볼까요?”
   - “무료로 시작하기” 버튼

스타일:
- 카드형 레이아웃
- 둥근 모서리
- 부드러운 그림자
- 여행 사진 중심
- 모바일 앱 느낌의 웹 UI
- 지도 핀, 루트 라인, 프로필 아바타, 하트/별 아이콘 활용
```

## 2. 스와이프 관광지 탐색 화면 UI 프롬프트

```text
여행지 카드를 스와이프하며 취향을 수집하는 웹 앱 화면을 디자인해줘.

사용자는 관광지 사진, 지역, 태그, 간단한 설명을 보고 좌우 스와이프로 관심 여부를 선택한다. 
일반 좋아요와 강한 선호를 표현하는 슈퍼라이크 기능이 핵심이다.

화면 구성:
1. 상단
   - 현재 여행 지역 선택 드롭다운: “강릉”, “전주”, “부산” 등
   - 진행률: “12 / 30개 장소 탐색 중”
   - 내 취향 분석 보기 버튼

2. 중앙
   - 큰 관광지 카드
   - 카드에는 고품질 여행 사진
   - 장소명
   - 지역
   - 태그: #감성카페 #바다뷰 #뚜벅이추천 #야경
   - 짧은 설명
   - 예상 체류 시간
   - 접근성 정보 아이콘

3. 하단 액션 버튼
   - 패스
   - 좋아요
   - 슈퍼라이크
   - 상세보기

4. 우측 사이드 패널
   - 지금까지 좋아요한 장소
   - 슈퍼라이크한 장소
   - AI가 추정한 내 여행 취향: 자연 42%, 카페 31%, 액티비티 18%

인터랙션 느낌:
- 카드가 살짝 기울어지며 넘어가는 애니메이션
- 슈퍼라이크 시 별빛 효과
- 좋아요 시 하트 애니메이션

스타일:
- 모바일 앱을 웹에 확장한 듯한 UI
- 이미지 중심
- 20~30대가 선호할 만한 감각적인 여행 플랫폼 분위기
- 깔끔하고 직관적인 정보 구조
```

## 3. 그룹 여행방 대시보드 UI 프롬프트

```text
친구들과 함께 여행 계획을 세우는 그룹 여행방 대시보드 UI를 디자인해줘.

이 화면은 그룹원들이 각자 스와이프한 결과와 슈퍼라이크한 관광지를 한눈에 확인하고, AI 추천 장소와 여행 준비 상태를 볼 수 있는 협업형 화면이다.

화면 구성:
1. 상단 헤더
   - 여행방 이름: “강릉 1박 2일 감성 여행”
   - 여행 날짜
   - 참여 인원 아바타
   - 초대 링크 버튼
   - 경로 확정하기 버튼

2. 좌측 패널
   - 그룹원 목록
   - 각 그룹원의 취향 태그
   - 스와이프 완료율
   - 슈퍼라이크 개수

3. 중앙 메인 영역
   - 그룹 취향 요약 카드
   - “우리 그룹은 바다뷰, 감성카페, 야경 장소를 선호해요”
   - 공통 추천 관광지 TOP 5
   - 구성원별 슈퍼라이크 장소

4. 우측 패널
   - AI 추천 장소
   - 추천 이유 표시
   - 예: “지훈님과 민서님이 모두 카페·바다뷰 장소를 선호해서 추천했어요”
   - 장소 저장 버튼
   - 지도에 추가 버튼

5. 하단
   - 여행 준비 단계 타임라인
   - 취향 수집 완료
   - 추천 장소 확인
   - 지도 경로 설계
   - 최종 루트 확정

스타일:
- 협업 툴과 여행 앱이 결합된 느낌
- Notion처럼 정돈된 정보 구조
- 지도, 카드, 프로필 아바타를 함께 활용
- 밝고 친근한 분위기
```

## 4. 지도 캔버스 기반 여행 경로 설계 UI 프롬프트

```text
그룹 여행자가 지도 위에서 함께 여행 경로를 그리고 수정하는 웹 서비스 UI를 디자인해줘.

이 화면은 단순 지도 저장 기능이 아니라, 지도 위를 캔버스처럼 활용해 장소를 연결하고 메모를 남기며 여행 동선을 설계하는 협업형 지도 화면이다.

화면 구성:
1. 전체 화면 지도
   - 관광지 핀
   - 그룹원이 슈퍼라이크한 장소는 멤버별 색상 핀으로 표시
   - AI 추천 장소는 반짝이는 추천 배지로 표시
   - 확정된 여행 경로는 굵은 라인으로 표시

2. 좌측 패널
   - 여행 일정: Day 1, Day 2
   - 방문 장소 리스트
   - 장소별 예상 체류 시간
   - 이동 시간
   - 순서 변경 드래그 앤 드롭

3. 우측 패널
   - 선택한 장소 상세 정보
   - 사진
   - 태그
   - 추천 이유
   - 누가 슈퍼라이크했는지
   - 메모
   - “경로에 추가” 버튼

4. 상단 툴바
   - 펜 도구
   - 경로 연결 도구
   - 메모 추가
   - 구역 표시
   - AI 경로 최적화
   - 공유하기

5. 협업 요소
   - 실시간으로 움직이는 멤버 커서
   - “민서님이 이 장소를 추가했어요” 알림
   - 장소별 댓글 또는 간단한 의견 남기기

스타일:
- 지도 기반 SaaS 느낌
- 여행 계획을 시각적으로 설계하는 인터랙티브 UI
- 깔끔한 패널 구조
- 경로 라인과 핀 컬러가 직관적으로 구분되도록 디자인
```

## 5. AI 추천 관광지 화면 UI 프롬프트

```text
사용자와 그룹의 여행 취향 데이터를 바탕으로 AI가 관광지를 추천하는 웹 앱 화면을 디자인해줘.

추천은 단순 인기순이 아니라, 스와이프, 좋아요, 슈퍼라이크 데이터를 기반으로 개인과 그룹 모두가 만족할 가능성이 높은 장소를 제안한다.

화면 구성:
1. 상단
   - 제목: “AI가 찾은 우리 그룹 맞춤 여행지”
   - 설명: “각자의 슈퍼라이크와 취향 태그를 분석해 추천했어요”
   - 필터: 자연, 카페, 맛집, 전시, 액티비티, 야경, 접근성

2. 추천 요약 카드
   - 그룹 취향 매칭률
   - 공통 만족도
   - 특정 멤버 강한 선호 여부
   - 이동 동선 적합도

3. 추천 장소 카드 리스트
   - 장소 사진
   - 장소명
   - 지역
   - 태그
   - 매칭 점수
   - 추천 이유
   - “지도에 추가”
   - “상세보기”
   - “패스”

4. 추천 이유 예시
   - “3명 중 2명이 바다뷰 장소를 슈퍼라이크했어요”
   - “카페와 산책 코스를 함께 선호하는 그룹 패턴과 잘 맞아요”
   - “현재 경로에서 12분 거리라 동선에 추가하기 좋아요”

5. 숨은 관광지 추천 섹션
   - 리뷰 수는 적지만 취향 매칭률이 높은 지역 중소 관광지
   - “새롭게 발견할 만한 장소” 배지

스타일:
- AI 추천이지만 어렵지 않고 친근하게 보이도록 디자인
- 추천 근거를 투명하게 보여주는 UI
- 카드형 레이아웃
- 데이터 시각화는 가볍고 직관적으로 표현
```

## 6. 여행 커뮤니티 피드 UI 프롬프트

```text
사용자가 여행 후기와 여행 경로를 공유하는 인스타그램형 여행 커뮤니티 피드 UI를 디자인해줘.

이 커뮤니티는 단순 사진 후기 게시판이 아니라, 실제 여행자가 만든 여행 루트와 방문 장소, 후기를 함께 공유하는 공간이다.

화면 구성:
1. 상단
   - 검색창: “강릉 카페 루트, 전주 하루 코스 검색”
   - 테마 필터: 감성카페, 뚜벅이, 비 오는 날, 가족여행, 야경, 로컬맛집
   - 글쓰기 버튼
   - 루트 공유 버튼

2. 메인 피드
   - 여행 사진 중심 카드
   - 작성자 프로필
   - 여행 제목
   - 지역
   - 여행 기간
   - 대표 태그
   - 좋아요, 저장, 공유 버튼

3. 루트 카드
   - “친구들과 다녀온 1박 2일 강릉 카페 루트”
   - 지도 미리보기
   - 방문 장소 개수
   - 총 이동 거리
   - 예상 소요 시간
   - “내 여행방에 가져오기” 버튼

4. 상세 게시물 화면
   - 사진 슬라이드
   - 여행 후기
   - 방문 장소 리스트
   - 실제 이동 경로 지도
   - 댓글
   - 저장한 사용자 수
   - 비슷한 루트 추천

5. 우측 사이드바
   - 인기 여행 루트
   - 이번 주 많이 저장된 지역
   - 로컬 가이드 추천 게시물

스타일:
- Instagram, Pinterest, 여행 매거진 느낌 결합
- 사진 중심
- 카드형 피드
- 저장하고 따라가고 싶은 실용적인 여행 콘텐츠 분위기
```

## 7. 전체 웹 서비스 통합 UI 프롬프트

```text
20~30대 그룹 여행자를 위한 취향 기반 여행 협업 플랫폼의 전체 웹 서비스 UI를 설계해줘.

서비스 핵심은 다음과 같다:
- 관광지 카드를 스와이프하며 개인 취향을 수집
- 슈퍼라이크한 장소를 그룹 여행방에서 공유
- 지도 캔버스 위에서 함께 여행 경로 설계
- AI가 개인과 그룹 취향을 분석해 관광지 추천
- 여행 후 여행기와 여행 경로를 커뮤니티 피드에 공유

필수 화면:
1. 랜딩 페이지
2. 온보딩 및 취향 선택 화면
3. 스와이프 관광지 탐색 화면
4. 그룹 여행방 대시보드
5. 지도 캔버스 경로 설계 화면
6. AI 추천 관광지 화면
7. 여행 커뮤니티 피드
8. 여행 루트 상세 페이지
9. 마이페이지

디자인 방향:
- 20~30대가 좋아할 만한 감각적인 여행 서비스
- 모바일 앱 경험을 웹으로 확장한 듯한 UI
- 스와이프, 지도, 카드, 피드 중심
- 그룹 협업이 잘 드러나는 프로필 아바타와 실시간 상태 표시
- 지역 중소 관광지와 숨은 관광지 발견을 강조
- AI 추천은 어렵지 않고 친근하게 표현

컬러:
- 메인 컬러: 코랄 오렌지 또는 선셋 핑크
- 서브 컬러: 민트, 라이트 블루, 크림 화이트
```
