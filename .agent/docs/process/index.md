# 프로세스 문서

Git Flow, 브랜치 ledger, 문서 운영처럼 작업 절차를 정의하는 문서를 보관합니다.

- `git_workflow.md`: Git Flow, commit convention, submodule 운영 정책.
- `branching_agent_docs.md`: 브랜치별 AI 문서 격리와 통합 규칙.
- `domain_development_policy.md`: 도메인 개발 전 설명, test-first, 검증 보고 정책.
- `javadoc_policy.md`: backend JavaDoc 한국어 작성 기준과 완료 조건.
- `backend_team_tdd_checklist.md`: 백엔드 3인 역할, 담당 디렉토리, TDD 진행, merge conflict 방지 체크리스트. 서비스 전체가 아니라 backend 완료 판단 기준입니다.
- 프론트 API 연동 현황은 `../exec-plans/active/2026-06-22-frontend-api-integration-workboard.md`에서 작업 ID, 상태와 브랜치/PR 기준으로 관리합니다.
